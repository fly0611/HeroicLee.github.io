异常介绍

当我们的程序发生一些解释器无法继续处理下去的事情，我们的解释器无法执行无法编译，并且会抛出错误(异常)
一般的异常是一些逻辑错误，语法错误，无法生成结果等
抛出错误(异常)之后，我们的程序将无法正常执行下去(抛出的错误会使我们的程序(一般是终止)做错误的默认处理)
但是我们也可以自己去改写出现错误之后的默认处理动作，也叫做捕获异常；这么做的目的就是为了提高我们程序的健壮性，应对各种复杂的互联网计算机环境
